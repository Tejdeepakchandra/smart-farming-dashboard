3.2 Functional Requirements

3.2.1 User Registration and Authentication (FR-01)

Inputs: Name, Email, Password, Confirm Password, Farm Name, Location

Processing: Validate fields, hash password (bcrypt), create user in MongoDB, create session.

Outputs: Redirect to Dashboard on success; inline error messages on failure.

Error Handling: Duplicate email, weak password, invalid credentials — all display user-friendly messages.

3.2.2 Crop Management — CRUD (FR-02)

Inputs: Crop Name (8 presets or custom), Field Name, Planting Date, Harvest Date, Status.

Processing: On CREATE — store crop and auto-generate 20 sensor readings. On READ — fetch with latest reading. On UPDATE — validate and save. On DELETE — remove crop and all associated data.

Outputs: Crop list with sensor snapshots; crop detail with ideal vs actual comparison.

3.2.3 IoT Sensor Data Simulation (FR-03)

Inputs: "New Reading" button (1 reading) or "Fill 24h Data" button (20 readings).

Processing: SensorSimulatorService looks up crop profile, generates values (70% within ideal range, 30% drift), stores readings, checks alert thresholds.

3.2.4 Real-time Dashboard (FR-04)

Processing: Fetch latest reading, compare against ideal range, determine status (Good/Low/High), calculate gauge percentage.

Outputs: 5 sensor cards with emoji, large value, farmer-friendly label ("Perfect"/"Too Cold"/"Too Hot"), gauge bar, and ideal range text. Color coding: Green=good, Amber=warning, Red=danger.

3.2.5 Smart Alert System (FR-05)

Processing: Low moisture (<60% of ideal_min), High temp (>115% of ideal_max), High humidity (>110% of ideal_max), Heavy rain (>200% of ideal_max). Cooldown: same alert type won't re-trigger for 2 hours.

Outputs: Alert stored with type (danger/warning/info), displayed with emoji severity, unread count badge.

3.2.6 AI-Powered Insights (FR-06)

Inputs: "Ask AI" button click.

Processing: Fetch latest reading, build prompt with sensor values + crop name + ideal ranges, send to Gemini API (gemini-2.0-flash), parse and store response.

Outputs: 3 actionable recommendations in AI panel. Error handling for API rate limits (HTTP 429) and connectivity issues.

3.2.7 AI Farming Chatbot (FR-07)

Inputs: Text message (max 1000 chars).

Processing: Build prompt with farming-assistant persona, send to Gemini API, store message+response in chat_messages collection.

Outputs: AI response in chat bubble with typing indicator. Quick-ask suggestion buttons for common questions.

3.5 Non-Functional Requirements

3.5.1 Performance

  • Dashboard shall load within 3 seconds on stable internet
  • Single sensor simulation shall complete within 1 second
  • Batch simulation (20 readings) shall complete within 5 seconds
  • Charts shall render within 2 seconds for up to 500 data points
  • AI insight generation within 30 seconds (dependent on Gemini API)

3.5.2 Reliability

  • Gemini API failures handled gracefully with user-friendly error messages
  • MongoDB connection failures show fallback error pages
  • Alert cooldown prevents duplicate alerts within 2-hour windows

3.5.3 Availability

  • 99% uptime when deployed on hosting platform
  • MongoDB Atlas provides 99.95% uptime SLA
  • Gemini rate limits handled with HTTP 429 error display

3.5.4 Security

  • Passwords hashed using bcrypt (12 rounds)
  • All routes protected by Laravel auth middleware
  • User data isolation via user_id ownership checks
  • CSRF token protection on all forms
  • MongoDB Atlas uses TLS encryption
  • API keys stored in .env (never exposed to frontend)

3.5.5 Maintainability

  • Laravel MVC architecture with separated Models, Views, Controllers
  • Business logic encapsulated in service classes (SensorSimulatorService, GeminiService)
  • Adding new crop type requires only adding one array entry
  • All views use Blade template inheritance for consistent layout

3.5.6 Portability

  • Runs on any server with PHP 8.2+ and MongoDB extension
  • Frontend uses CDN-loaded libraries — no build step required
  • Browser-agnostic (Chrome, Firefox, Edge, Safari)
  • Responsive design for mobile, tablet, and desktop

3.7 Design Constraints

  1. Must use Laravel 12 with MVC architecture
  2. Must use MongoDB via mongodb/laravel-mongodb package
  3. Must use Laravel Breeze for authentication
  4. Must use Gemini API free tier (no paid AI services)
  5. No physical IoT hardware — simulation only
  6. Must use Blade templating (no SPA frameworks)

3.9 Other Requirements

  • Must work without terminal commands after deployment
  • Auto-generate 20 initial readings when a new crop is created
  • Charts must be interactive (hover tooltips, clickable legend)
  • Landing page publicly accessible without authentication
