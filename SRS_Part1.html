1. Introduction

1.1 Purpose

This Software Requirements Specification (SRS) document provides a complete description of the Smart Farming Dashboard with IoT Integration web application. It defines all functional and non-functional requirements, system architecture, data flow, and interface specifications. The intended audience includes the project evaluator/faculty, development team, and stakeholders.

1.2 Scope

Product Name: Smart Farming Dashboard with IoT Integration

The software will:

  • Provide a web-based dashboard for farmers to monitor crop health using simulated IoT sensor data (temperature, soil moisture, humidity, light intensity, rainfall)
  • Support crop-specific sensor profiles — each of 8 crop types (Rice, Wheat, Tomato, Corn, Potato, Sugarcane, Cotton, Soybean) has unique ideal sensor ranges
  • Simulate IoT data via the web interface — no terminal commands required for deployment
  • Compare sensor readings against ideal ranges and show farmer-friendly labels like "Too Hot", "Needs Water", "Perfect"
  • Provide AI-powered recommendations using Google Gemini API
  • Offer an AI chatbot for natural language farming questions
  • Generate automatic alerts for drought, heat stress, humidity, and heavy rainfall
  • Visualize sensor trends using interactive Chart.js line graphs

The software will NOT: Connect to physical IoT hardware (simulation only), provide weather forecasts, or support multi-language localization.

2. General Description

2.1 Product Perspective

The Smart Farming Dashboard is a standalone web application designed as a semester project. It simulates an IoT-based farming environment where sensors generate environmental data, a dashboard displays health indicators, an AI engine provides crop-specific recommendations, and an alert system monitors for dangerous conditions.

2.3 User Characteristics

Primary User: Farmer — May have limited technical knowledge. Needs simple visual indicators instead of raw data. Uses mobile phones or basic laptops. Expects plain language like "Needs Water!" instead of "Below ideal moisture threshold".

Secondary User: Agricultural Advisor / Teacher — Has technical understanding of farming parameters. Evaluates the project for academic assessment.

2.4 General Constraints

  • No physical IoT hardware — uses simulated sensor data
  • Gemini API free tier: ~15 requests/minute rate limit
  • Requires internet for MongoDB Atlas and Gemini API
  • Requires modern browser (ES6+, CSS3)
  • English-only interface

2.5 Assumptions and Dependencies

Assumptions: Modern browser available; PHP 8.2+ on server; MongoDB Atlas accessible; Gemini API key valid.

Dependencies: MongoDB Atlas (M0 free tier), Google Gemini API, Laravel 12, Composer, Node.js/npm, Tailwind CSS CDN, Chart.js CDN.
